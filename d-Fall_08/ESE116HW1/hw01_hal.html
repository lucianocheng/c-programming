Overview:

In this first assignment, you will contribute to the development of HAL, the innocent calculator that someday evolves to rule humanity. Your contribution to the project will exercise your skills in basic C function construction, C integers and characters, the use of if statements, function return values, and switch/case statements.

The file hal.c contains two functions that you will have to complete and turn in. These functions are currently blank:

  • int get_user_input()
    • Prompts a user for the operands and operators necessary to run a calculation
      • The documentation above the get_user_input() function in the hal.c file contains detailed instructions on the input format we expect.
      • In order for the program to loop properly, the function must return correct values. Please fill out the function as specified.
      • You will not be dealing with terminal input directly, we have wrapped the terminal input into the functions get_integer_from_terminal() and get_character_from_terminal() which we have provided for you.
  • int calc_result(int, int, char)
    • Takes two operands and an operator, and calculates the result accordingly
      • Documentation for the expected output of calc_result() is included in the hal.c file, above the calc_result() function. Please fill out the function as specified.
      • You are required to use a switch statement to complete this function.

The file hal_main.c contains four functions you must utilize to write the hal.c functions. Do not modify these functions, you must use them in their current form:

  • int main(int argc, char* argv[])
    • The main function does the following:
      • Prints a welcome message to the terminal
      • Loops, and calls your get_user_input() function for each loop, until get_user_input() returns USER_EXIT
      • Prints a goodbye message
      • Exits
  • void print_result(int, int, char, int)
    • The print_result() function gets called by your get_user_input() function.
    • It prints the calculation using printf()
  • int get_integer_from_terminal()
    • The get_integer_from_terminal() function gets called by your get_user_input() function.
    • It retrieves one integer value from the terminal, and returns this integer
  • char get_character_from_terminal()
    • The get_character_from_terminal() function gets called by your get_user_input() function.
    • It retrieves one character value from the terminal, and returns this character

The following definitions are in hal.h:

  • USER_EXIT is defined as 0
    • Use USER_EXIT when asked in the function description
    • Do not use 0 directly
  • USER_CONTINUE is defined as 1
    • Use USER_CONTINUE when asked in the function description
    • Do not use 1 directly

To compile your code to a binary executable called 'hal', type in the following at the command line, assuming your hal.c, hal_main.c, and hal.h files are in your current working directory:

   gcc -Wall hal_main.c hal.c -o hal

The '-Wall' flag will print out all warnings that the compilation generates. It is highly recommended that you eliminate warnings in your coding, as warnings can often lead to undefined behavior (bugs) in your programs when you run them.

After executing the code above, the following line can be used to execute your program:

   ./hal
