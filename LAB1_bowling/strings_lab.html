ESE 113/116 Lab: String Exercise

The purpose of this lab is to create a simple program that parses input from STDIN, breaks up the input into words, deletes repeated words, and prints out the words and their sizes to STDOUT

Goals:

  • Writing a program to read from STDIN and write to STDOUT
  • Using strlen, strcmp, strcpy from the string library, as well as constructing strings from characters
  • Using getchar(), a simple function that reads one byte (one character) from STDIN at a time

Reference:

  • C Strings: Background
  • getchar() man page
  • Standard Streams in Wikipedia

Exercise: String Data Printer

Please read the C Strings: Background page before starting this exercise. Unless you fully grasp the concepts presented in the background page, this exercise cannot be completed correctly.

Download and complete strings.c. Extended instructions for completion are included in comments in the file. Once completed, this program will read a stream from STDIN, break up the stream into words, delete repeated words, and print out the words and their sizes to stdout.

Specific sections of code have been left blank for you to fill in. See the instructions in the file.

For reading from STDIN it is sufficient to use only getchar() and not scanf.

The purpose of the getWord() function is to write a single word from STDIN to newString. The word should have no spaces or newlines preceeding it or following it. Please keep this overarching purpose in mind when constructing your code.

An example run and interaction might look like this:

$ ./strings
hello
'hello', size: 5
hello
goodbye
'goodbye', size: 7
fox
'fox', size: 3

To facilitate testing, a text file test.txt has been provided that covers all of the corner cases. To run your strings program with the test.txt file, you must redirect the test.txt file into the program. You do this with the '<' on the command line:

./strings < test.txt

COMMON POINT OF CONFUSION: The getWord() function should always return 1 as long as a word was found. GetWord() should return 0 if and only if getchar() returned EOF and NO WORD WAS FOUND BEFOREHAND.

In other words, if a word was found, and the word was immediately followed with an EOF, getWord() should still return 1. The NEXT call to getWord() will then return 0.
